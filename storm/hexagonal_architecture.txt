클린 아키텍쳐 & 헥사고날 아케텍쳐

데이터 입력 - 포트 - 어뎁터 - 어플리케이션(비즈니스로직) - [모델(도메인)] - 어플리케이션(비즈니스로직) - 어뎁터 - 포트 - 데이터 저장/전달
           (Handler)     (Service)               (Models)      (Service)          (Repository)

데이터 입력의 포트가 사용 가능한 어뎁터
  HTTP Handler (REST API)
  GrapeQL Handler
  gRPC Handler
  WebSocket Handler
  ...

  어뎁터를 통해서 데이터를 수신하여 도메인을 통해서 어플케이션으로 전달 데이터 전달


  어플리케이션에서 필요한 로직을 처리하고

  다시 도메인을 통해서 값을 전달하여 저장 / 전송 / 내부 전달

  메인에서
  
  0. 레파지토리에서 사용할 db 생성
  1. db를 담은 저장용 레파지토리 생성
  2. 레파지토리 담은 서비스 생성
  3. 서비스 담은 헨드러 생성
  4. 헨들러에 있는 메소드 함수를 웹프레임워크 라우팅에 연결

  핸들러에서 데이터를 받으면 서비스를 호출
  서비스에서 비즈니스 로직 처리후 레파지토리 호출
  레파지토리에서 데이터 자정/전송


  - 인터페이스 이용

  데이터베이스 연결
  레파지토리 생성 (db)
  데이터레이어 (레파지토리)
  서비스레이어 (데이터레이어)
  컨트롤러레이어 (서비스레이어)
